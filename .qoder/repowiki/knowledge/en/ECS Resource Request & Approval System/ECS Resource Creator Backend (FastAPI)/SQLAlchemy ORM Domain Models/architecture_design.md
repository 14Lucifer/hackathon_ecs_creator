Each file defines one SQLAlchemy `Base`-derived ORM class plus a small `str`-based `enum.Enum` (e.g. `UserRole`, `RequestStatus`, `AuditAction`) that maps to a database-level enum via `Enum(..., name=...)`. All classes inherit from `app.database.Base` and declare columns with PEP 695-style `Mapped[...] = mapped_column(...)` annotations; relationships use `relationship("...", lazy="joined")` for eager loading. Cross-model foreign keys are declared directly on fields (`ForeignKey("users.id")`, etc.), so there is no separate relationship table — `UserSession.user`, `ResourceRequest.user`, and `ResourceRequest.template` are the only explicit back-references. Business invariants live as module-level constants (`MAX_TEMPLATES = 6`, `ACTIVE_STATUSES`, `MAX_ACTIVE_REQUESTS_PER_USER`, `ENCRYPTED_KEYS`) alongside the model they constrain, and `__init__.py` re-exports every public symbol through `__all__` so callers import from `app.models` rather than individual files. No business logic or persistence methods are defined on the models themselves — they are pure schema + lightweight computed properties (e.g. `ResourceRequest.is_active`).
