Three cooperating modules form the application root:
- `config.py` defines a Pydantic `BaseSettings` (`Settings`) with `.env` loading and an `@lru_cache`-wrapped `get_settings()` accessor consumed by every other module.
- `database.py` builds a single `sqlalchemy.create_engine` from `Settings.database_url`, exposes a module-level `SessionLocal` session factory and a `get_db()` generator dependency used by routers for request-scoped DB access; all ORM models subclass the exported `DeclarativeBase`.
- `main.py` is the FastAPI entry point: it registers a `lifespan` context manager that calls `seed_admin()` (creates the first admin account via `hash_password` + `SessionLocal`), then mounts eight domain routers (`auth`, `users`, `templates`, `requests`, `approvals`, `active_resources`, `audit`, `settings`) under the `/api` prefix and exposes a `/api/health` probe. Dependency direction is one-way: `main` → `config`/`database`/`routers`/`services.password`; `database` → `config`; no cross-imports between routers at this level.
