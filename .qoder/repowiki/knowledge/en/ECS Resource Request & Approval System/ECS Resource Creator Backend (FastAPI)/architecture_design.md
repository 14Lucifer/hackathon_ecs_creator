The `app/` package is split into layered sub-packages — `models` (SQLAlchemy ORM), `schemas` (Pydantic v2 contracts), `routers` (FastAPI route handlers), `services` (business logic + Alibaba Cloud SDK wrappers), plus shared `middleware`, `config.py`, `database.py` and `main.py`. `main.py` instantiates the FastAPI app, mounts routers under `/api`, registers middleware, and seeds the initial admin user; all routers import from `models`, `schemas`, and `services` rather than from each other, keeping feature boundaries isolated. Cross-cutting concerns are centralized: `auth_users` provides the login/profile flow and a dependency-injected current-user resolver reused by every other router; `settings_audit` reads/writes system settings and writes immutable entries to the audit log; `requests_approvals` orchestrates the request→approval lifecycle and calls `aliyun_services` (ECS/VPC wrappers) to provision or delete instances. Database access goes through the shared engine/session in `database.py`, with schema evolution handled by Alembic (`alembic.ini`, `alembic/env.py`, `versions/`). The top-level `Dockerfile`, `entrypoint.sh`, and `requirements.txt` provide the containerized runtime.
