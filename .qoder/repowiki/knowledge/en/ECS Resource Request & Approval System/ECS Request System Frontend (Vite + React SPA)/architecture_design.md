The frontend is a Vite-built SPA whose entry (`src/main.jsx`) mounts `App` into `index.html`. `App.jsx` owns the single source of truth for authentication via a local `AuthContext` (created here and consumed by child pages) and declares all routes with `react-router-dom`: `/login`, `/portal` (user), and `/admin/*` (admin sub-app). Role checks in the router guard each route, so page components never perform auth themselves. The shared UI primitives (`Spinner`, badges, modals, toasts) live in `components/ui.jsx` and are imported uniformly by both user and admin pages. All HTTP calls go through the singleton `services/api.js` client, which reads the session from cookies and emits a `session-expired` event that `App` listens on to force logout — this is the only cross-cutting concern besides routing.
