- `alembic.ini` declares script location as `alembic/`, sets logging to WARN/INFO on stderr, and defers `sqlalchemy.url` to runtime.
- `env.py` is the Alembic entry point: it reads `DATABASE_URL` via `app.config.get_settings()`, imports `app.database.Base` and `app.models` (side-effect import registers all ORM classes), then configures either offline or online migration execution through `context.configure` with `target_metadata = Base.metadata`.
- `versions/0001_initial_schema.py` is the sole revision; it defines three `sa.Enum` types (`user_role`, `request_status`, `audit_action`) and creates the core tables (`users`, `ecs_templates`, `resource_requests`, `audit_logs`, `settings`, `sessions`) with explicit foreign keys and indexes. Downgrade drops tables in reverse creation order and explicitly drops enums via `op.get_bind()`.
- Dependency direction is one-way: this module depends on `app.config`, `app.database`, and `app.models`; nothing in `app/` imports from `alembic/`.
