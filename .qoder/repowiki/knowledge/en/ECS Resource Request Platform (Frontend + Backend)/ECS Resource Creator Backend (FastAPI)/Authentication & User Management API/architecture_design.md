Two FastAPI `APIRouter` instances mounted under `/auth` and `/users` implement the auth/user surface. `routers/auth.py` exposes stateless cookie-session endpoints (`/login`, `/logout`, `/me`) that delegate session creation/destruction and current-user resolution to `app.middleware.auth`. `routers/users.py` is gated by a global `Depends(require_admin)` dependency so every route requires an admin role; it provides standard CRUD plus two Excel helpers using `openpyxl` (template download and `.xlsx` batch upsert). `schemas/auth.py` defines the Pydantic request/response models (`LoginRequest`, `SessionUser`) used across both routers. Dependency direction: routers → middleware (session + authorization) → services (password hashing/verification) → SQLAlchemy models via `get_db`; schemas are pure data contracts with no import on domain code.
