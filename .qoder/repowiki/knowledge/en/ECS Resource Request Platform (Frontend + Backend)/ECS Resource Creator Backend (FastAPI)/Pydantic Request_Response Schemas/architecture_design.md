Flat package of Pydantic `BaseModel` subclasses organized by domain (auth, user, template, request, approval, audit, settings) with a single `__init__.py` re-exporting every public model via `__all__`, so callers import from `app.schemas` rather than individual files. Each domain file follows an input/output split: create/update schemas accept fields with `Field(...)` validators, while `*Out` schemas expose read-only responses and set `class Config: from_attributes = True` to map directly onto SQLAlchemy ORM rows. Cross-file references are kept minimal — `request.RequestOut` imports `template.TemplateOut` as a nested response, but there is no circular dependency; the rest of the module is self-contained and has no runtime dependencies beyond Pydantic and standard library types.
