The SPA entry `src/main.jsx` mounts `App.jsx`, which is the single orchestrator: it creates the `AuthContext` (the only source of authentication state), declares all routes via `react-router-dom` (`/login`, `/portal`, `/admin/*`), and applies role-based guards at the route level so child pages never handle auth themselves. The user portal lives under `pages/user/` and the admin sub-app under `pages/admin/`; both consume shared UI primitives from `components/ui.jsx`. All HTTP traffic goes through the singleton `services/api.js`, which reads cookies for session handling and emits a `session-expired` event that `App` listens on to force logout — this is the sole cross-cutting concern besides routing.
