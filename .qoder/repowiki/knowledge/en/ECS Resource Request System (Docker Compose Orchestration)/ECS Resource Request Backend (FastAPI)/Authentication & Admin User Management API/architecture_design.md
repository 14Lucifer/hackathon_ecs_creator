Two `APIRouter` instances mounted under `/auth` and `/users`. `routers/auth.py` exposes stateless cookie-session endpoints (`/login`, `/logout`, `/me`) that delegate session creation/destruction and current-user resolution to `app.middleware.auth`; it queries the `User` ORM model directly via `get_db` and verifies passwords through `app.services.password.verify_password`. `routers/users.py` is gated at the router level with `dependencies=[Depends(require_admin)]` so every route requires an admin role; it provides standard CRUD plus two Excel helpers using `openpyxl` (template download and `.xlsx` batch upsert) and enforces deletion blockers (admin cannot be deleted/disabled, active resource requests must be resolved first). `schemas/auth.py` defines the shared Pydantic contracts `LoginRequest` and `SessionUser` used across both routers. Dependency direction: routers → middleware (session + authorization) → services (password hashing/verification) → SQLAlchemy models via `get_db`; schemas are pure data contracts with no import on domain code.
