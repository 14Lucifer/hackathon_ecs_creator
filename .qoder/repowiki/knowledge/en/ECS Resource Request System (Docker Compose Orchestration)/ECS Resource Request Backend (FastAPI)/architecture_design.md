The `app/` package is organized as a layered FastAPI application: `models` (SQLAlchemy ORM), `schemas` (Pydantic v2 contracts), `routers` (route handlers), and `services` (business logic + Alibaba Cloud SDK wrappers) are wired together by `main.py`, which instantiates the app, mounts routers under `/api`, registers middleware, and seeds the initial admin user. Feature boundaries are enforced by a one-way dependency rule â€” routers import from models/schemas/services but never from sibling routers, so cross-feature communication flows exclusively through service functions. Cross-cutting concerns are centralized in `auth_users` (cookie-based login/logout/me plus a current-user dependency injected into every other router), `settings_audit` (system settings storage and immutable audit-log writes), and `requests_approvals` (the requestâ†’approval lifecycle that calls `aliyun_services` to provision/delete instances and manage DNS). Database access goes through the shared engine/session in `database.py`, with schema evolution driven by Alembic (`alembic.ini`, `env.py`, `versions/`). The top-level `Dockerfile`, `entrypoint.sh`, and `requirements.txt` provide the containerized runtime.
