The root `docker-compose.yml` defines four services with strict startup ordering: `postgres:15-alpine` is health-checked via `pg_isready`, then `backend/` (FastAPI) depends on it, then `frontend/` (Vite build container) produces a static `/output` volume, and finally `nginx:alpine` serves that volume while proxying all `/api/*` traffic to the backend. Nginx is the only externally exposed port (80); there is no direct networking between frontend and backend containers. Both application containers receive secrets exclusively through `.env` loaded via `env_file`, keeping credentials out of images and code.
