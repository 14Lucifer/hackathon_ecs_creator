Two FastAPI `APIRouter` instances (`/settings`, `/audit`) are mounted under the app's router tree; both declare `dependencies=[Depends(require_admin)]` so every endpoint is gated by the admin middleware. The `/settings` router delegates persistence to `app.services.settings_service.get_setting` / `set_setting`, which transparently encrypts/decrypts values in `ENCRYPTED_KEYS` via `app.services.crypto` before hitting the `SystemSetting` SQLAlchemy model, keeping secrets out of plaintext responses by masking them with a constant `MASK`. The `/audit` router is read-only: it builds a parameterized `select(AuditLog)` statement, optionally filters on `AuditAction` and orders by `created_at asc/desc`, returning `AuditLogOut` Pydantic models. Cross-cutting concerns (DB session via `get_db`, password hashing/verification, session invalidation) are imported from sibling `app.database`, `app.middleware.auth`, `app.models`, `app.schemas`, and `app.services.password` packages — this module owns only HTTP boundaries and thin orchestration.
